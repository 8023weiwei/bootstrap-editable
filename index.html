Editable for Bootstrap In-place editing with Bootstrap Form and Popover

About

Invokes in-place editing feature to any element of your page

  • based on Bootstrap form and popover
  • supported types: text, textarea, select, date
  • validation on client and server-side
  • keyboard events (escape/enter)

Download stable (v1.0.5)

Usage and markup

You can activate editable on your page easily with special markup and a few line of javascript. Options can be defined via either data-* attributes or json options.

Text and textarea

Just set data-type, data-pk, data-name and data-url of <A> element. 

<a href="#" id="username" data-type="text" data-pk="1" data-name="username" data-url="post.php" data-original-title="Enter username">superuser</a>
$('#username').editable();

Alternatively, you can set all the same options via javascript

<a href="#" id="username">superuser</a>
$('#username').editable({
                           type: 'text',
                           pk: 1,
                           name: 'username',
                           url: 'post.php',  
                           title: 'Enter username'
                        });
Tip! Usually URL for submit is the same for all fields. You can define it in defaults: $.fn.editable.defaults.url = 'post.php';

Select

For 'select' element additionally set data-source and data-value

<a href="#" id="group" data-type="select" data-name="group" data-pk="1" data-value="5" data-source="groups.php" data-original-title="Select group">Admin</a>
$('#group').editable();

Date

Date is based on jQuery UI datepicker with Bootstrap theme. You can set data-format to control format of date

<a href="#" id="dob" data-type="date" data-format="dd.mm.yy" data-placement="bottom" data-pk="1" data-name="dob" data-original-title="Select Date of birth">15.05.1984</a>
$('#dob').editable();

Options

List of all possible options

Name type default description
type string 'text' input type - text | select | textarea | date
name string null name of field, required. If not set, will be taken from ID attribute.
pk string | function null primary key of editable object (e.g. record id in database). Use function to get id dynamically, e.g.:
function() {
    return $('#user_id').text();
}
value string $(element).text() initial value. Should be defined for 'select' type to store selected ID
params object null additional params to submit
url string null url to submit data. Request contains next fields: pk, name, value and params
emptytext string 'Empty' text shown on empty elements
toggle string | jQuery null element that toggles popover by click. Can be CSS selector or HTML. if null - text itself used as toggle. E.g. data-toggle="#pencil"
send string 'ifpk' strategy for sending data on server - ifpk | always | never. 'ifpk' = data will be sent only if pk defined
enablefocus bool true wether to return focus on element after popover is closed. It's more functional, but focused links may look not pretty
validate function | object null client-side validation. If function returns string - means validation not passed and string showed as error. E.g.: 
validate: function(value) {
  if($.trim(value) == '') 
    return 'This field is required';
}
Definition as object is usefull when applying editable to several elements and difference only in validation. Object structure is name: function, e.g. 
validate: {
         username: function(v) {...},
         fullname: function(v) {...}
      }
success function null called after success submit. If returns string - means error occured and string showed as error. Parameters equals to $.ajax() success callback. E.g.: 
function(data) {
    if(typeof data == 'object' && !data.success) return data.msg;  
}
error function null called on ajax error when submitting data. If returns string - it will be shown as error message. Parameters equals to $.ajax() error callback. E.g.: 
function(xhr) {
    if(xhr.status == 500) return 'Internal server error';  
}
Tip! If you need one success/error callback for all fields you can define it in defaults: 
$.fn.editable.defaults.success = function(data) { ... }

Type specific options

There are additional options for different input types

Select

Name type default description
source string | object null source for data in select element. If string - considered url for ajax call to load items, otherwise should be json. E.g. data-source="{'M': 'male', 'F': 'female'}"
autotext bool false if true element's content will be overwritten by text according to value (during initialization)
prepend string | object false allows to prepend values in select list.
E.g. prepend: 'Not set' - new value {'': 'Not set'} will be automatically inserted at the begining of select list

Date

Name type default description
format string yy-mm-dd date format. It overwrites datepicker's dateFormat
datepicker object {
dateFormat: 'yy-mm-dd',
changeMonth: true,
changeYear: true
}		 configuration for jQuery UI datepicker.

Methods

$().editable(options)

Initializes an editable.


Below methods are mainly for creating new records (details in next section)

$().editable('validate')

Calls validate method for one or several editable elements. Returns object with names and validation messages, e.g.:

{
username: "username is requied",
fullname: "fullname should be minimum 3 letters length"
}

$().editable('getValue')

Returns values for one or several editable elements, e.g.:

{
username: "superuser",
fullname: "John"
}

$().editable('markAsSaved')

Mark editable as saved to server (removes class "editable-changed", etc).

Events

No events defined for this component.

Creating new record

When creating new record it's primary key (record id) is undefined and in-place-editing has no related object on backend. The approach is to store values in html until record will be created. After then editable works as usual. For that reason there is send = 'ifpk' option. You can create hidden element to store ID and point to it in related editable fields, e.g.:

<span style="display: none" id="user_id"></span>
$('.myeditable').editable({
    url: 'post.php',
    send: 'ifpk',
    pk: '#user_id', //link to hidden element that stores ID as inner text 
    validate: {
        username: function(v) {if(v == '') return 'Username is required!'}
    } 
});

Initially #user_id is empty - no requests will be submited to server, values will be stored in html and marked with css class editable-changed

When you click on create user button, validation will run for all values and if correct - submit to server. To perform these actions easily there are three methods validate, getValue and markAsSaved
E.g. 

$('#save-btn').click(function() {
       var errors = $('.myeditable').editable('validate'); //run validation for all values
       if($.isEmptyObject(errors)) {
           var data = $('.myeditable').editable('getValue'); //get values
           $.post('new.php', data, function(response) {
              $('#user_id').text(response.id); 
              $('.myeditable').editable('markAsSaved'); //mark fields as saved
              ...
          }); 
       } else {
          var msg = 'Validation errors! ';
          $.each(errors, function(k, v) { msg += v+"<br>"; });
          $('.alert-error').html(msg).show(); 
          ...
       }
});
